## Official Definition
* **Vulnerable Software Vector:** A threat vector that exploits weaknesses, bugs, or flaws in software applications, operating systems, or firmware to gain unauthorized access or execute malicious code.

## Key Characteristics/Technical Details
* **Zero-day Exploits:** Exploiting a vulnerability that is unknown to the vendor and has no available patch.
* **Buffer Overflow:** Sending more data to a memory buffer than it can hold, allowing for code execution or system crashes.
* **Unpatched Software:** Exploiting known vulnerabilities for which a patch exists but has not been applied by the user or organization.
* **Legacy Systems:** Systems that are no longer supported by the vendor and do not receive security updates.
* **Default Configurations:** Exploiting software that has been left with insecure default settings (e.g., default passwords).

## Real-World Examples
* **Equifax Breach (2017):** Exploited a known vulnerability in the Apache Struts web framework that had not been patched.
* **WannaCry (2017):** Exploited the EternalBlue vulnerability in Windows (SMB protocol) for which a patch had been released but not widely applied.

## Exam Tips
* **Keywords:** Exploit, Zero-day, Unpatched, Buffer overflow, Legacy, Vulnerability scanner.
* **Scenario:** An attacker "uses an exploit kit" against a server running an "outdated version" of software.

## Relationship to other concepts
* **Vulnerabilities:** This vector is the active exploitation of the vulnerabilities described in the Vulnerabilities section.
* **Mitigation:** Patch management, vulnerability scanning, and application whitelisting.
