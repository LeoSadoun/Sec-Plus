### Official Definition
Alternative security measures implemented when a primary control is either not feasible, not cost-effective, or has failed, providing a similar level of protection.

### Key Characteristics
- **Alternative:** A "Plan B" or substitute.
- **Context-Specific:** Tailored to bridge a specific security gap.
- **Temporary or Permanent:** Used until a primary control can be implemented or as a permanent workaround.

### Real-World Examples
1. **Secondary Log Monitoring:** Using intense log review because an old application cannot support modern encryption.
2. **Security Guard:** Stationed at a door because the electronic lock is broken.
3. **Virtual Patching:** Using a WAF to block exploits for a vulnerability that cannot be patched immediately.
4. **Manual Sign-in:** Using a paper log when a badge reader system is offline.
5. **Air-Gapping:** Physically isolating a system because it cannot be adequately secured on the network.

### Exam Tips
- **Keywords:** "Alternative," "Substitute," "Workaround," "Feasibility," "Gap-filler," "Instead of."
- **Pitfall:** Compensating controls must provide *comparable* protection, not just *some* protection.
- **Interaction:** These are often audited heavily to ensure they meet the security requirement of the missing primary control.

### Relationship to other controls
- **Technical:** A WAF (Technical) can compensate for a lack of application patching.
- **Physical:** A Guard (Physical/Operational) can compensate for a failed lock.
