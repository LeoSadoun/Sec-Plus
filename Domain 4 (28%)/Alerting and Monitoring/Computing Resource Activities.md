## Official Definition
The specific events and metrics generated by hosts (servers, workstations) that indicate security or performance status.

## Key Mechanisms/Tools
*   **CPU/RAM/Disk I/O:** Metrics that indicate system load; spikes can indicate DoS, malware (crypto-mining), or system failure.
*   **Event Logs:** 
    *   **Security Logs:** Success/failure logins, account changes, privilege escalation.
    *   **System Logs:** Driver failures, service starts/stops.
    *   **Application Logs:** App-specific errors or activities.
*   **Process Monitoring:** Tracking which executables are running and which ports they are listening on.
*   **File Integrity Monitoring (FIM):** Monitoring critical system files for unauthorized changes (e.g., `C:\Windows\System32` or `/etc/passwd`).

## Real-World Examples
*   A monitoring tool detects an unknown process `malware.exe` starting up on a domain controller.
*   FIM triggers an alert when the `hosts` file on a web server is modified, potentially indicating a DNS hijacking attempt.

## Exam Tips
*   **Keywords:** FIM, event logs, process monitoring, utilization metrics.
*   **Scenario:** How to detect unauthorized changes to system files? **FIM**.
*   **Scenario:** High CPU usage on a server with no users logged in? Investigate for malware or background tasks.

## Relationship to other concepts
*   **Hardening:** Monitoring ensures that the security settings applied during hardening remain in place.
*   **Vulnerability Management:** Unusual activity can be a sign that a known vulnerability is being exploited.
*   **Data Sources:** Computing resource activities are one of the primary "Data Sources" for security investigations.
