## Official Definition
The technologies and processes used to identify, analyze, and report on security events and system health in real-time.

## Key Mechanisms/Tools
*   **SIEM (Security Information and Event Management):** Centralized platform that collects and correlates logs from across the enterprise (e.g., Splunk, Sentinel, ELK).
*   **SOAR (Security Orchestration, Automation, and Response):** Tools that take SIEM alerts and execute automated "playbooks" (e.g., automatically blocking an IP after 5 failed logins).
*   **SNMP (Simple Network Management Protocol):** Used to monitor the status of network hardware.
*   **NetFlow/IPFIX:** Protocols used to monitor the *flow* of network traffic (who is talking to whom) rather than the packet content.
*   **Dashboarding Tools:** Visual interfaces (like Grafana) used to display monitoring data.

## Real-World Examples
*   A SIEM triggers an alert when it detects a "Brute Force" pattern: 10 failed logins followed by 1 successful login from a foreign IP within 5 minutes.
*   Using NetFlow to identify a sudden spike in outbound traffic to a known malicious command-and-control (C2) domain.

## Exam Tips
*   **Keywords:** SIEM, SOAR, correlation, NetFlow, SNMP, playbooks.
*   **Scenario:** Need to correlate logs from multiple different sources? Use a **SIEM**.
*   **Scenario:** Need to *automatically* respond to a common alert? Use **SOAR**.

## Relationship to other concepts
*   **Log Data:** Monitoring tools are the consumers of the data generated by system and application logs.
*   **Incident Response:** Alerting tools are the primary "Detection" mechanism in the IR lifecycle.
*   **Computing Resource Activities:** These tools provide the "visibility" into what is happening on the hosts.
